1.FastJson概述

在学习FastJson的安全漏洞之前，我们首先得了解一下什么是FastJson？

Fastjson是阿里巴巴的开源JSON解析库，它可以解析 JSON 格式的字符串，支持将 Java Object 序列化为 JSON 字符串，也可以从 JSON 字符串反序列化到 Java Object

主要是两个接口来分别实现序列化和饭序列化操作
JSON.toJSONString 将java对象转化为json对象，也就是序列化的过程

JSON.parseObject/JSON.parse 将json对象重新转化为java对象，顾名思义反序列化的过程

以往的序列化和反序列化的过程中间都是序列化数据在承担运载数据的作用，但是在FastJson中就是用特殊的json数据承载这个任务了
还需要注意的一点是，在FastJson中并不是所有的java对象都能够被转化为JSON，只有javabean格式的对象才能够转化为JSON

  • get方法,想要能够获取到get方法,必须是map,AtomicBoolean,AtomicInteger,AtomicLong这四个类型,并且传入参数值必须是0

FastJson反序列化漏洞攻击利用总结

首先需要提出，FastJson的序列化不是调用到的原生的序列化，这个是他自己根据JSON数据实例化出来的对象
两者的相同点都是最后会执行恶意方法或者调用静态代码块，进而执行恶意代码

类比原生反序列化，FastJson的反序列化采用它自己的反序列化器，一个入口点在readObject，一个就在setter或者getter
明确目的之后肯定就是一番寻找，这里我们先给出结论，再看有没有可能自己总结出寻找的理由。

所以要能够当作FastJson的攻击链部分需要满足如下条件
如果是Setter

  1. 非静态函数
  2. 返回类型为void或者当前类
  3. 参数个数为1

如果是Getter

1.非静态方法
2.无参数
3.返回类型必须是（或继承自）Collection或Map或AtomicBoolean或AtomicInteger或AtomicLong

上述条件都是最后获取构造器的时候三段for循环中的内容，也就是第一段for循环以及第三段for循环，能否获取到指定set或者get方法条件，如果不能获取，那还提什么调用呢？
